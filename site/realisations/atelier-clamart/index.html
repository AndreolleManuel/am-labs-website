Le contexte

Avant : papier, Excel et téléphone.

L'Atelier Clamart est une structure d'ateliers d'arts créatifs basée en région parisienne (Clamart, 92140 + un 2e lieu) avec plusieurs professeurs, plusieurs catégories de cours (baby, enfants, ados, adultes), des cours hebdomadaires récurrents, des stages pendant les vacances, et des événements ponctuels.

Avant la plateforme : cahier papier pour le planning, tableur Excel pour le suivi des participants, réservations par téléphone et SMS, croix cochées à la main pour la présence. Conséquences : créneaux double-réservés, croisements de paiement par chèque/espèces difficiles à tracer, communication parent fastidieuse, équipe sous tension administrative.

La solution

Une plateforme complète, pas un outil de réservation générique.

Plateforme web sur mesure livrée en mars 2026, qui couvre l'intégralité du métier de l'atelier, côté clients comme côté équipe. Elle est utilisée quotidiennement par les clientes (réservations, crédits, paiement) et par l'équipe (planning, gestion, communication, finance).

Architecture : front Next.js 14 + back Express, base PostgreSQL, paiement Stripe, mails Brevo, hébergement Vercel + Render, SMS optionnel via Twilio.

Modules livrés

Tout le métier dans une interface.

01

Réservation cliente

Catalogue des cours, planning hebdomadaire visible, réservation en 2 clics, gestion d'invités (parents qui réservent pour enfants).

02

Crédits par catégorie d'âge

Chaque tranche d'âge a ses propres crédits, ses formules, ses dates d'expiration. Alertes auto avant expiration.

03

Paiement Stripe

Achat de crédits + réservation directe dans la même transaction. CB, Klarna en plusieurs fois, factures auto, remboursement conditionnel sous 12h.

04

Planning hebdo + saisons

Vue semaine type, calendrier mois, gestion des saisons (rentrée, vacances scolaires zone A/B/C), wizard de génération automatique.

05

Liste d'attente intelligente

Place libérée → notification automatique au 1er de la file avec 2h pour confirmer. Sinon passage au suivant.

06

Back-office admin complet

Dashboard KPIs, gestion clients, formules, profs, lieux. Stats financières (CA, ventes par formule). Multi-rôles admin/prof/client.

07

Multi-professeurs

Plusieurs profs par cours, attribution selon disponibilités, vue planning par prof, demandes de séances par les profs eux-mêmes.

08

Mails transactionnels

Confirmation de réservation, rappel avant cours, expiration crédits, invitation client par token sécurisé, planning du lendemain aux profs.

Stack technique détaillée

Sous le capot.

Frontend : Next.js 14 (App Router) + React + TypeScript + Tailwind CSS, FullCalendar pour la vue planning admin, design system custom (palette teal + terracotta).

Backend : Node.js + Express + TypeScript, ~2 500 lignes de code, 15 tables PostgreSQL, authentification JWT en cookies HttpOnly avec CSRF double-submit.

Sécurité : validation server-side de tous les prix, webhook Stripe en transaction PostgreSQL avec re-vérification du montant (anti-manipulation), security headers (CSP, X-Frame-Options), contraintes CHECK en base.

Intégrations : Stripe (paiement + Klarna + facturation invoice-first), Brevo (emails transactionnels + newsletter), API gouvernementale pour la sync automatique des vacances scolaires.
